## Why this matters

`@biomejs/biome` is already a devDependency of `packages/plugin` but there is
**no `biome.json`, no `lint`/`check` script, and no CI lint step** — it's dead
tooling. Adding a minimal Biome config plus a CI gate gives the project a fast,
single-tool lint+format check (Biome 2.x is a viable ESLint+Prettier
replacement for a small TS lib: https://biomejs.dev/guides/getting-started/).
The goal is a *non-mutating check in CI*, not a mass reformat of the codebase.

## Steps

### Step 1: Decide the Biome version and create the config

Prefer bumping `@biomejs/biome` to `^2` in `packages/plugin/package.json`
devDependencies, then run a **non-frozen** `bun install` to update the lockfile.
(If the operator wants to avoid the bump, configure for the existing `^1.9.4`
instead and skip the version edit — note which path you took.)

**Lockfile gotcha:** the repo's normal install command (and CI) use
`bun install --frozen-lockfile`. If you bump the biome version, you MUST run a
plain `bun install` (NOT `--frozen-lockfile`) so `bun.lock` is regenerated, then
commit the updated `bun.lock`. Otherwise the very first verify command and CI
will fail with a frozen-lockfile mismatch.

**Verify (after the bump path):** `bun install --frozen-lockfile` → exit 0 (this
proves `bun.lock` is now consistent with the bumped manifest).

Run `cd packages/plugin && bun x @biomejs/biome init` to generate `biome.json`.
Edit it to:
- Enable the formatter and linter with **recommended** rules only (no custom rule
  zoo on the first pass).
- Scope `files.includes` (Biome 2.x) / `files.include` (1.x) to `src/**` and
  optionally `tests/**` and `tests-extended/**`. **Exclude** `dist/**`,
  `node_modules/**`, and `**/*.d.ts` (generated declaration files).
- Match the repo's existing style so the formatter doesn't churn the file:
  inspect `src/index.ts` — it uses **2-space indent** and **single quotes**. Set
  `formatter.indentStyle: "space"`, `indentWidth: 2`, and
  `javascript.formatter.quoteStyle: "single"`.

**Verify**: `cd packages/plugin && bun x @biomejs/biome check src` runs and
reports its findings (exit code may be non-zero before fixes — that's expected
here).

### Step 2: Bring the source to a clean state without changing behavior

Run `cd packages/plugin && bun x @biomejs/biome check --write src` to apply
**safe** fixes (formatting + safe lint fixes only). Inspect the diff: it must be
formatting/whitespace/import-ordering only. If Biome wants an **unsafe** fix or a
behavioral change, do NOT apply it — instead relax that specific rule to `"warn"`
or `"off"` in `biome.json` and note it for follow-up.

**Verify**:
- `cd packages/plugin && bun run build` → exit 0 (still compiles).
- `cd packages/plugin && bun run test` → all pass (behavior unchanged).
- `cd packages/plugin && bun run test:unit-extended` → all pass.

### Step 3: Add scripts

In `packages/plugin/package.json` scripts add:
- `"check": "biome check src"` (non-mutating; CI uses this)
- `"lint": "biome lint src"`
- `"format": "biome format --write src"`

**Verify**: `cd packages/plugin && bun run check` → exit 0 (clean after Step 2).

### Step 4: Add a CI lint job

In `.github/workflows/test.yml` add a `lint` job mirroring the existing jobs'
setup (checkout, setup-bun, setup-node, cache, `bun install --frozen-lockfile`),
then a single step: `cd packages/plugin && bun run check`. A single Node version
is sufficient for lint (e.g. `24`) — no need for the full matrix. Make the
existing `unit`/`unit-extended`/`harness` jobs independent of it, or have them
`needs: lint` if you want lint to gate (operator preference — default: make
`lint` a standalone job that runs in parallel, not a blocker, to avoid slowing
the matrix).

**Verify**: the workflow YAML is valid —
`bun x js-yaml .github/workflows/test.yml >/dev/null` (or
`python3 -c "import yaml,sys; yaml.safe_load(open('.github/workflows/test.yml'))"`)
→ exit 0.
